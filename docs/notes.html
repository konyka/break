GPU Timers

4 GPU timestamp queries: shadow, forward, scene (total), postfx
GL: glQueryCounter(GL_TIMESTAMP) | VK: VkQueryPool(VK_QUERY_TYPE_TIMESTAMP) + timestampPeriod

Render Scale

4 presets: 0.3x, 0.5x (default), 0.75x, 1.0x
Dynamic FBO recreation on resize. Upscale via blit at pipeline end.

Shadow Cascades (CSM)

4 cascades with split scale (F4). Debug viz (F1 mode 4) colors: R/G/B/Y.
Pipeline: depth-only pass → cascade_vp[4] uniforms → PBR shader samples cascade map.
Shadow bias adjustable: u_shadow_bias uniform (Z/X keys, default 0.002).

Frustum Culling

Method: AABB-based frustum culling using mesh bounding boxes.
AABB computation: computed from vertex positions at mesh load time.
World-space test: 8 local AABB corners × world_transform → world AABB → frustum_test_aabb
Scope: forward scene nodes only (skinned meshes excluded).
Debug UI: shows Culled: N per frame.

Terrain System

Grid: 64×64, 40 units wide, 1.5x height scale
Heightmap: mutable f32[64*64] array, initialized from sin(x*0.3)*cos(z*0.3)*2 + sin(x*0.1+z*0.15)*3
Splatmap: procedural multi-texture based on height + slope (4 biomes)
• Grass: flat terrain (slope < 0.5), green with fine pattern
• Rock: steep terrain (slope > 0.3), gray with noise grain
• Snow: high altitude (y > 3.5), white with sparkle
• Sand: low altitude (y < 1.5), tan with grain texture
Shader: custom terrain.vert/frag (+ _vk variants), with Blinn-Phong lighting
Editing: terrain_modify_height(t, wx, wz, radius, strength) — quadratic falloff brush
VBO rebuild: only affected region + 1-cell border (partial glBufferSubData)
Height query: bilinear interpolation via terrain_get_height
Keys: Y=raise, H=lower (hold), Q=terrain follow, brush target = camera + forward*5

Water Plane

Mesh: 80×80 unit quad at y=-1.0 (adjustable)
Rendering: alpha-blended, rendered in scene pass after terrain
Shader: custom water.vert/frag (+ _vk variants)
Visuals: 3-layer sine wave animation, Schlick Fresnel, specular highlights
Depth fade: distance-based (10-50 units), transparency 0.5-0.85
Color: deep blue (0.1, 0.3, 0.5), surface lighter with wave modulation
Keys: = toggle, (/) adjust water level ±0.5

Underwater Effect

Trigger: camera.position.y < water.water_y
Clear color: deep blue (0.0, 0.05, 0.15) replaces background
Debug UI: shows [UNDERWATER] indicator and water level
Scene save/load: water_y and water.enabled persisted

Terrain Shadows

Method: cascade 0 shadow map lookup in terrain.frag
Uniforms: u_light_vp (cascade VP matrix), u_shadow_map (depth texture), u_shadow_bias
Shader: terrain_shadow() projects world pos to light space, single-tap depth compare
Result: 0.4 shadow factor (60% darkened) or 1.0 (lit)
Fallback: returns 1.0 when UV out of shadow map bounds

Water Shadows

Method: same cascade 0 shadow lookup as terrain
Shader: water_shadow() in water.frag / water_vk.frag
Factor: 0.5 (50% darkened) in shadow, 1.0 lit
Binding: shadow map at binding 1, light VP as uniform mat4

Entity Duplication

Key: ] — duplicate selected entity

Physics: half_extent (0.5,0.5,0.5), mass 1.0, dynamic
Counter: selected_entity_count incremented

Shoreline Foam

Location: terrain.frag / terrain_vk.frag — where terrain height approaches water level
Shader: smoothstep(0.8, 0.0, dist) * (1.0 - smoothstep(-0.3, -0.8, dist))
Pattern: modulated with sin(x*6)*sin(z*5) for wave-like foam
Color: mixes toward (0.85, 0.9, 0.95) — white-blue foam
Uniform: u_water_y passed from terrain_render()

Collision Counter

Field: PhysicsWorld.collision_count (reset each physics_step)
Display: shown in debug UI line: "Collisions: N"
Counted: every AABB overlap resolved in physics_step()

Underwater Terrain Caustics

Shader: terrain.frag / terrain_vk.frag — u_time uniform
Effect: when fragment below water level (world_y < u_water_y), add caustic pattern
Pattern: sin/cos dual-wave pow(,3) sharpened + depth modulation for underwater darkening
Uniform: loc_time in terrain.h/terrain.c, passed total_time from main.c
Intensity: fades with depth below surface, multiply by (1.0 - depth * 0.3)

Camera Path Record/Playback

Key: , — 3-state cycle: start recording → stop recording → playback → stop playback
Buffer: cam_path[600] stores {px, py, pz, yaw, pitch} per frame
Record: every frame stores camera position + orientation, auto-stops at 600 frames
Playback: overrides camera_update(), loops path continuously
Debug UI: [REC] or [PLAY] indicator with frame counter

Terrain Crater Preset

Key: ; — now cycles 5 presets (added Craters)
Algorithm: noise base + 5 procedural craters at fixed positions
Crater shape: depth falloff (1 - d/r) * 0.6 * height_scale, rim at r < d < r*1.5
Crater positions: (0.3,0.7), (0.7,0.3), (0.5,0.5), (0.2,0.2), (0.8,0.8) in normalized coords
Radii: 0.08 + i*0.02 per crater (slightly varied sizes)

Collision Screen Shake

Trigger: any new physics collision (collision_count increase between frames)
Intensity: adds 0.3 per collision, clamped at max 1.5
Effect: sinusoidal projection matrix perturbation (47.3Hz + 31.7Hz), decays 0.92×/frame
Existing: P-key boom also triggers shake at 1.0, explosion at 2.0

Entity Respawn on Fall

Trigger: entity position Y < -20.0 (fell off world)
Respawn: teleported to (0, 5, 0) with zero velocity
Scope: all entities with CTransform + CRigidBody in physics-sync loop
Sync: resets both CTransform.pos and physics body position+velocity

Explosion Force

Key: 1 — radial impulse explosion from camera position
Force: max(0, 20 - distance×2) along direction away from epicenter
Effects: all physics bodies pushed + 5000 particle burst + screen shake 2.0
Direction: vec3_normalize(body_pos - camera_pos) scaled by force magnitude

Sun Disk Halo

Shader: skybox.frag / skybox_vk.frag
Sun disc: smoothstep(0.9995, 0.9999, cos_sun) — sharp core
Sun halo: smoothstep(0.993, 0.999, cos_sun) — wider soft glow
Intensity: disc 0.05 × SUN_INTENSITY, halo 0.008 × SUN_INTENSITY, both × extinction

Entity Magnet

Skips: static physics bodies
Effect: inverse-distance attraction, bodies accelerate toward camera

Velocity Damping Mode

Key: 9 — now cycles 3 modes: gravity → zero-g → damping
Damping: velocity *= 0.95 per frame for all non-static bodies

Terrain Erosion Brush

Key: A — now cycles 3 modes: raise/lower → flatten → erode
Algorithm: thermal erosion — 4-neighbor height comparison
Formula: material moves from high to low neighbors, proportional to slope
Rate: amount = min(max_diff, h*0.5) * 0.5 per iteration
Function: terrain_erode(Terrain *t, f32 wx, f32 wz, f32 radius, i32 iterations)
Debug UI: [ERODE] indicator when active

System Kinetic Energy

Formula: total_KE = sum(0.5 × mass_i × speed_i²) across all physics bodies
Display: "System KE: N J" only when > 0.01

Center of Mass & Momentum

Center of mass: mass-weighted average position: sum(pos_i × mass_i) / total_mass
Momentum: sum(mass_i × velocity_i) across all bodies
Display: "Center of mass: (x,y,z)" and "Momentum: (x,y,z) |p|=mag"
Condition: only shown when physics bodies > 1

Terrain Brush Radius Indicator

System Mass & Average Speed

Total mass: sum of all body masses, shown with KE as "mass: N kg"
Average speed: mean velocity of bodies with speed > 0.1 m/s
Moving count: "Avg speed: N m/s (X/Y moving)" — X bodies in motion out of Y total

Physics Body Spatial Bounds

Computation: min/max position across all non-zero bodies
Span: diagonal distance of bounding box = vec3_len(max - min)
Display: "Bodies span: N bounds: (min)-(max)"
Note: useful for seeing how spread out entities are

Terrain Height Standard Deviation

Formula: std = sqrt(sum((h_i - avg)²) / N) — 2-pass computation

Architecture Notes

RHI Abstraction

Files: rhi.h, rhi_gl.c, rhi_vk.c
Resources: Slot-based with generation counters (handle = index + generation)
Key APIs:
rhi_pipeline_create — vertex layout auto-detected from desc flags
rhi_cmd_set_uniform_* — no push constants, uniform-by-name
rhi_gpu_timer_create/begin/end/elapsed_ms — both backends
rhi_screenshot — GL: glReadPixels + BMP, VK: stub
rhi_set_vsync — GL: glXSwapIntervalEXT, VK: swapchain recreate
wireframe — desc bool → GL glPolygonMode, VK POLYGON_MODE_LINE
rhi_buffer_update_region — partial VBO update (offset+size) for terrain editing

Mat4 / Vec3 Convention

Mat4: Column-major: e[col][row]
Vec3: e[3] array accessor, no named members
rhi_cmd_draw: (cmd, vertex_count, instance_count) — 3 args only

Build Commands

OpenGL:
cd engine && cmake -B build && cmake --build build

Vulkan:
cd engine && cmake -B build-vk -DENGINE_VULKAN=ON && cmake --build build-vk

Tests:
./build/test_vulkan (runs on both backends despite name)

Physics ↔ ECS Sync

Every frame after physics_step(): ECS query for Transform+RigidBody components
Copies physics->bodies[id].position → CTransform.pos[3]
Ensures instanced rendering displays physics bodies at correct positions
E key spawns full ECS entity (Transform + RigidBody + MeshRef) for proper rendering
